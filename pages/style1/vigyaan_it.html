Problem Statements

1. ALUMINI SITE DEVELOPMENT

The main aim of this problem statement is to generate and receive ideas for the development of our colleges� alumni site. You are to present workable ideas which will enhance the look as well as the functionality of the site. The idea presented will be judged based on creativity and effectiveness.

2. NOTIFICATION SYSTEM"

You have to create a notification system that sends alerts to both students and faculty, along with necessary data. Your system should allow the transference of notices, study material and assignments given by faculty. You have to create your own network and your own push notification. You will be judged on the efficiency of your model, the precautions taken to avoid problems, and assumed cost of your model's implementation.

3. AVERAGE TRAFFIC VELOCITY FINDER"

You have to calculate average traffic velocity at a particular region, at a particular time where luminosity is invariant, with it working in all conditions, like overcast, sunny, etc. Data should be taken directly from net. The data should be analyzed, and the average velocity should be stored in a database.

4. PLACEMENT ANALYSING TECHNOLOGY

There is a lot, a student of our college can gain if he is provided with statistics on placements and companies. This system should take in data from our placement cell and analyze trends that can be spotted, in order to prepare students as to what to expect when he or she plans to sit for that company. The system should provide facts about the company, what qualities it prefers and where our seniors generally strike out. Means a complete requirement analysis of the company. Data should be taken from internet. Student performance analysis should be done. Representation should be in Box chart, histogram and pie chart. Student data should be taken from database.

5. ONLINE EXAMINATIONS

You have to create your own online exam module with UI, where at a time, your module should be able to handle more than 100 users easily with servers. You have to set up the whole network. Admin panel should be created where faculty can enter the questions. Choose backend of your choice. Student should be able to login with his credentials. Result should be stored in database and should be shown after exam.

6. MAP BUILDING VIA ROBOT

In this problem statement, you are required to create a software system to map the paths taken by a robotic vehicle, so as to develop maps in uncharted areas. The IT department will provide the robot to the selected participants. The map should be created as the robot captures the road via a camera mounted on it. A demo of robot will also be given to interested students.

7. STUDENT INFORMATION CENTRE

Take four servers, one web, one mail, and two DNS servers, of which, one is primary, and other secondary. Create a site named www.nitraipur.com with any IP address. Site should open via both domain name and IP address. In site, you have to create a child domain about college department, notes, assignment, solution and online test. Create a login page and a registration page where user address should be reflected inside Linux server. Means password should also reflect in shadow file. User groups should be reflected in Linux server, i.e. teacher�s group and student group should be separate and users in teacher group should be able to access student group while vice-versa not possible.This should be handled by admin panel. A feedback section should be made for students. Admin panel for teachers should made. This site should be able to be accessed from any computer of that network. A firewall should be there for network such that it blocks connection with Telnet, SMTP, SSH and FTP ports.

A team can choose only one problem statement and in case a team opts for technical domain, they have to choose it of their department only.
Team size can be maximum of 3 members.
All participating teams have to prepare an abstract describing the solution and submit it in IEEE format as a PDF file as given in sample abstract by 21/09/2014, 11PM at abstract@aavartan.in .
Along with abstract PDF, you also have to submit another PDF file with the Team details( Name, College, Branch, Sem, Contact, Email ID) of all the team members.
Late Submission won't be entertained."
